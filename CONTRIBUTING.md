# Contributing to DES Winter School 2026

Thank you for your interest in contributing to **DES Winter School 2026**!  
We appreciate every contribution, big or small, and your efforts help improve the project for everyone.

## How to Contribute

There are several ways you can contribute to **DES Winter School 2026**:

### 1. Reporting Issues

- If you find a bug or have a feature request, please open an issue.
- Include detailed steps to reproduce the problem and any relevant information, such as the environment or version.

### 2. Suggesting Enhancements

- If you have ideas for improving **DES Winter School 2026**, please describe them in a new issue.
- Include examples or use cases whenever possible.

### 3. Submitting Pull Requests

- Fork the repository and create a branch for your changes.
- Keep your changes focused on a single purpose.
- Write clear and descriptive commit messages.
- Include tests for any new functionality.
- Ensure your code follows the existing style and conventions of **DES Winter School 2026**.

### 4. Code Review

- Review open pull requests when possible.
- Provide constructive feedback to help maintain the quality of **DES Winter School 2026**.
- Be respectful and collaborative in discussions.

## Guidelines

- Make sure all tests pass before submitting a pull request.
- Document any new features or changes.

## Thank You!

Your contributions make **DES Winter School 2026** better for everyone.  
Even small fixes, documentation updates, or suggestions are highly valued.  
We look forward to your contributions and appreciate your help in making **DES Winter School 2026** a stronger, more reliable project.
